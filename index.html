1) Cloud Resource Allocation and Scaling

Cloud providers handle Cloud Resource Allocation and Scaling through a combination of Policies, Mechanisms, Scaling, and Scheduling.

Policies:

  • Resource Check:
    • The system checks if enough resources are available before starting a task.
    • If resources are available, the task is accepted; otherwise, it is rejected.
  • Resource Allocation:
    • Distributes workloads across multiple servers to prevent overloading.
    • Improves performance and avoids bottlenecks.
  • Load Balancing:
    • Load Balancing Algoirhtm is a technique for Distributing the workloads across multiple servers.
    • Types of Load Balancing Algorithms:
      • Static Load Balancing Algorithm
      • Dynamic Load Balancing Algorithm
  • Power Saving:
    • Runs tasks on fewer servers to save power.
    • Puts idle servers in sleep mode.
    • Reduces power consumption and cost.
  • QoS:
    • QoS stands for Quality of Service
    • key metrics include:
      • Throughput:No. of Tasks completed per second
      • Latency:Time Delay between request and response
      • Response Time: Time taken to respond to a request

Mechanisms:

  • Control Theory:
    • Monitors system performance continuously.
    • Uses feedback to adjust resources automatically.
  • Machine Learning:
    • Predicts workload changes using past data.
    • Automatically adjusts resources as needed.
  • Performance vs Cost:
    • Resources are allocated to maximize performance with minimal cost.
  • Market-Based Allocation:
    • Uses pricing or auctions to distribute resources.
    • Users "buy" resources as per their needs.

Scaling:

  • Scalability:
    • The ability of a computing system to handle the workloads by adjusting its performance and costs is called Scalability.
    • Instead of upgrading the entire system, Cloud System allows you to add more servers to increase scalability.
  • Dynamic Scaling:
    • Scale Up: Add resources when CPU usage is high.
    • Scale Down: Remove resources when CPU usage is low.

Scheduling:

  • Resources like CPU time and bandwidth are shared among virtual machines using scheduling algorithms.
  • Common Algorithms:
    • Round Robin Algorithm
    • Shortest Job First Algorithm

2) Control Theory

Algorithms used in Control Theory:

  • Feedback Control Algorithm:
    • Continuously monitors the system performance.
    • Uses feedback to automatically adjust the resources.
  • PID Control:
    • PID stands for Proportional-Integral-Derivative.
    • The PID controller adjusts the system input based on the error value.
  • Dynamic Threshold:
    • Scale Up: Add resources when CPU usage is too high.
    • Scale Down: Remove resources when CPU usage is too low.
  • Two-Level Control Architecture:
    • It uses two controllers:
      • Application-level controller
      • Cloud-level controller

Control Theory Provides:

  • Scalability
  • Reliability
  • Performance
  • Responsiveness

3) Two-Level Resource Allocation Architecture

  • It uses two controllers:
    • Application-level controller
    • Cloud-level controller

Application-level Controller:

  • It manages Local Resources.
  • It allocates the resources based on the demand.

Cloud-level Controller:

  • It manages Global Resources.
  • It allocates the resources based on the app request.

Stability in a Two-Level Resource Allocation Architecture:

  • Stability depends upon the coordination between the Application-Level Controller and Cloud-Level Controller.

Feedback Mechanism:

  • Continuously monitors the system performance.
  • Uses feedback to automatically adjust the resources.

Dynamic Threshold:

  • Scale Up: Add resources when CPU usage is too high.
  • Scale Down: Remove resources when CPU usage is too low.

4) Start Time Fair Queuing Scheduling Algorithm

  • It is a hierarchical CPU scheduling algorithm used in cloud computing.
  • It fairly shares CPU time across different virtual machines.
  • It mainly consists of three types of nodes:
    • Root Node: Processor (CPU)
    • Intermediate Node: Applications and Virtual Machines
    • Leaf Node: Threads

Virtual Machines:

  • VM1
  • VM2

Applications:

  • A1
  • A2
  • A3

Example:

  • If VM1 is idle, its CPU bandwidth is automatically distributed to VM2.
  • Within VM2, if one of its threads becomes idle, its CPU bandwidth is automatically distributed to other threads.

5) Application Performance Managers (APM)

  • APM stands for Application Performance Managers.
  • Different APMs handle different resources such as CPU, Memory, Bandwidth, etc.
  • Several mechanisms are used for coordination between different APMs.

Mechanisms:

i) Hierarchical Coordination:

  • APMs are organized in a hierarchical structure.
  • Low-Level APMs manage subsystems like CPU, Memory, Bandwidth, etc.
  • High-Level APMs manage the overall system.
  • Coordination happens through:
    • Top-Down: Higher levels send policies to lower levels.
    • Bottom-Up: Lower levels send feedback to higher levels.

ii) Message Passing Protocol:

  • Message passing protocol is used to share information.
  • Common protocols include:
    • SOAP
    • REST
    • XML-Based messaging protocol

iii) Publishâ€“Subscribe Mechanism:

  • This mechanism is based on an event-driven model.
  • Managers subscribe to the events.
  • When a manager detects a change, they publish an update.
  • Other managers can react to the update.

iv) Middleware Layer:

  • The middleware layer acts as an interface for communication between APMs.
  • Communication between APMs happens using APIs or service buses.
  • Middleware acts as a bridge to connect different types of managers.
